## 数据模型

### 1. GitHub 账户表 (`github_accounts`)

| **字段名**         | **数据类型** | **约束 / 默认值** | **说明**                       |
| ------------------ | ------------ | ----------------- | ------------------------------ |
| **id**             | Integer      | 主键，自增        | 本地账户记录 ID                |
| **user_id**        | Integer      | 唯一外键          | 关联你系统主体的用户 ID        |
| **github_user_id** | BigInt       | 唯一，非空        | GitHub 侧用户的唯一 ID         |
| **login**          | String       | 非空              | GitHub 登录名 (例如 `octocat`) |
| **avatar_url**     | String       | 可空              | GitHub 头像地址                |

### 2. 仓库绑定表 (`repository_bindings`)

存储用户当前激活同步的那个唯一仓库，以及承载 Webhook 的配置。

| **字段名**          | **数据类型** | **约束 / 默认值** | **说明**                     |
| ------------------- | ------------ | ----------------- | ---------------------------- |
| **id**              | Integer      | 主键，自增        | 绑定的唯一标识               |
| **user_id**         | Integer      | 外键              | 关联主用户                   |
| **repository_id**   | BigInt       | 非空              | GitHub 仓库的唯一数字 ID     |
| **repository_name** | String       | 非空              | 仓库全称 (例如 `owner/repo`) |
| **repository_desc** | String       | 可空              | 仓库描述                     |
| **status**          | Boolean      | 非空              | 当前是否绑定为该仓库         |
| **bound_at**        | DateTime     | 默认当前时间      | 建立绑定的时间

### 3. Issue  (`issues`)

用于存储 GitHub Issue 的核心客观信息。

| **字段名称**          | **类型**     | **说明**                            |
| --------------------- | ------------ | ----------------------------------- |
| **id**                | Integer      | 主键，本地唯一标识                  |
| **repository_id**     | BigInt       | issue 所属的仓库 ID                 |
| **issue_id**          | BigInt       | GitHub 侧该 issue 的唯一 ID         |
| **Title**             | String       | issue 的标题                        |
| **Desc**              | Text         | issue 的描述正文                    |
| **labels**            | JSON / Array | issue 包含的标签集合                |
| **state**             | String       | issue 的状态（如 `open`, `closed`） |
| **milestone_id**      | BigInt       | 绑定的里程碑 ID                     |
| **mileston_title**    | String       | 绑定的里程碑标题                    |
| **github_created_at** | DateTime     | 在 GitHub 上的创建时间              |
| **github_updated_at** | DateTime     | 在 GitHub 上的更新时间              |
| **deadline_time**     | DateTime     | 截止时间                            |

### 4. Issue 响应表 (`issue_responses`)

用于记录针对 Issue 产生的所有后续动态（如 Comment、关联的 PR、Review 等）。

| **字段名称**          | **类型** | **说明**                                                    |
| --------------------- | -------- | ----------------------------------------------------------- |
| **id**                | Integer  | 主键，本地响应记录唯一标识                                  |
| **issue_id**          | BigInt   | 关联表 1 的 `issue_id`                                      |
| **Type**              | String   | 响应此 issue 的类型（如 `comment`, `PR`, `review`）         |
| **response_id**       | BigInt   | GitHub 侧该条响应（如具体的 comment ID 或 PR ID）的唯一标识 |
| **content**           | Text     | 响应内容正文                                                |
| **github_created_at** | DateTime | 在 GitHub 上的创建时间                                      |
| **github_updated_at** | DateTime | 在 GitHub 上的更新时间                                      |

### 5. PR 响应表 (`pr_responses`)

| **字段名称**          | **类型** | **约束 / 默认值** | **说明与业务用途**                                           |
| --------------------- | -------- | ----------------- | ------------------------------------------------------------ |
| **id**                | Integer  | 主键，自增        | 本地明细记录唯一标识。                                       |
| **issue_id**          | Int      | 非空，外键        | Issue的唯一 ID，将 PR 动态挂载到具体的任务上。               |
| **pr_id**             | Int      | 非空              | GitHub 侧该 PR 的唯一标识                                    |
| **pr_state**          | String   | 可空              | **PR 整体状态**：记录该动作发生时，或该动作导致 PR 变为了什么状态（如 `open`, `closed`, `merged`）。 |
| **review_state**      | String   | 可空              | **评审状态**：仅当 `type` 为 `review` 时有效（核心值：`APPROVED`, `CHANGES_REQUESTED`，用于触发主任务状态自动切换）。 |
| **type**              | String   | 非空              | **动作类型**：区分具体的动态（如 `comment` 评论, `commit` 新增代码提交, `review` 评审判定, `review_request` 请求评审）。 |
| **Content**           | Text     | 可空              | **内容**：评论正文、评审意见说明，或者是 Commit 的 Message 信息。 |
| **github_created_at** | DateTime | 非空              | （补充字段）该动作在 GitHub 上的实际发生时间。               |
| **github_updated_at** | DateTime | 非空              | （补充字段）该动作在 GitHub 上的最后修改时间。               |
