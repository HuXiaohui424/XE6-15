# 设想
形态：桌面展示
理由：桌面类似与一种启动台或者说入口，它的曝光量最大。


## 首次使用——初始化
创建github app。目的：提权及信息来源。
【可选】设置阶段目标——【可选】智能分析该阶段目标需要产出的内容。
获取分配给该账号的issue -> 初始化历史任务 -> 任务状态可以根据issue的状态、该账号是否有关联PR、发表评论、标签、操作日志等来确定任务状态。


## 后续
出现新的issue以及状态发生变化时，第一时间通过webhook更新任务（类似与微信消息通知，展示/通知的方式不同）。


## 其他说明
1.任务不是直接展示issue以及状态的更新，而是先通过Agent，然后展示。
2. Issue不一定作为唯一信息来源，PR、projec-board等都可以作为信息来源。
流程：github -> agent -> 前台
