# Scheduled Task & Audio Player · 接口文档

> **设计原则**：定时任务是一个通用调度框架，不绑定任何具体业务。事件（Action）独立定义、独立扩展。播放音频是当前已实现的第一个事件类型，并且可被外部直接调用。

---

## 1. 系统架构概览

```
┌─────────────────────────┐
│    外部调用者            │
│  (App 内任意模块)        │
└─────┬──────────────┬────┘
      │              │
      │  直接调用     │  管理定时任务
      ▼              ▼
┌───────────┐  ┌───────────────────┐
│ 音频播放器 │  │  定时任务管理器    │
│AudioPlayer│  │ScheduledTaskManager│
└───────────┘  └────────┬──────────┘
                         │ 触发时 ▼
                  ┌──────────────────┐
                  │   事件分发器       │
                  │  ActionDispatcher │
                  └────────┬─────────┘
                           │ 根据 ActionType 路由
              ┌────────────┼────────────┐
              ▼            ▼            ▼
        AudioPlay    FutureAction  FutureAction
         Action          A             B
```

- **AudioPlayer**：独立播放器，外部可直接调用
- **ScheduledTaskManager**：定时任务 CRUD，与具体事件解耦
- **ActionDispatcher**：内部路由，定时器触发时根据类型分发事件

## 2. 音频播放器 · `AudioPlayer`

独立模块，不依赖定时任务系统，任意模块可直接调用。

### 2.1 `play`

播放音频。

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `audioData` | `ByteArray` | 是 | 音频二进制数据 |
| `mimeType` | `String` | 否 | 音频格式，默认 `audio/mpeg`。支持：`audio/mpeg`、`audio/wav`、`audio/ogg`、`audio/aac` |
| `onComplete` | `(() -> Unit)?` | 否 | 播放结束回调，在主线程回调 |

**行为说明**：将二进制写入临时缓存文件，通过 `MediaPlayer` 播放。播完自动清理临时文件。若当前有音频正在播放，新的调用会中断前一个并立即播放新音频。

### 2.2 `stop`

立即停止当前播放。

### 2.3 `isPlaying`

查询当前是否正在播放，返回 `Boolean`。

## 3. 定时任务管理器 · `ScheduledTaskManager`

### 3.1 数据结构

#### `ScheduledTask`

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | `String` | 任务唯一 ID，创建时自动生成 |
| `triggerAtEpochMs` | `Long` | 触发时间戳（毫秒），基于 `System.currentTimeMillis()` |
| `actionType` | `ActionType` | 触发时执行的事件类型（枚举，可扩展） |
| `actionPayload` | `Map<String, String>` | 事件参数（key-value） |
| `label` | `String` | 任务备注 / 标签 |
| `isEnabled` | `Boolean` | 是否启用，`false` 时不会被触发 |
| `repeatIntervalMs` | `Long?` | 重复间隔（毫秒），`null` 表示不重复。最小 60_000（1 分钟） |
| `createdAtEpochMs` | `Long` | 创建时间戳 |

#### `ActionType`（枚举）

| 枚举值 | 说明 |
|---|---|
| `AUDIO_PLAY` | 播放音频（当前已实现） |
| `NOTIFICATION` | 发送本地通知（预留） |
| `CUSTOM` | 自定义（预留扩展） |

> 新增事件类型只需：1) 加一个枚举值，2) 在 `ActionDispatcher` 注册对应处理器，定时任务模块不需要改动。

---

### 3.2 API

#### 3.2.1 `createTask` — 创建定时任务

- 返回 `Result<ScheduledTask>`，成功时包含自动生成的 `id`

`CreateTaskRequest`：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `triggerAtEpochMs` | `Long` | 是 | 触发时间戳，必须大于当前时间至少 60 秒 |
| `actionType` | `ActionType` | 是 | 事件类型 |
| `actionPayload` | `Map<String, String>` | 否 | 事件参数。具 |
| `label` | `String` | 否 | 备注标签，默认空串 |
| `repeatIntervalMs` | `Long?` | 否 | 重复间隔，最小 60_000。`null` 为一次性任务 |

#### 3.2.2 `updateTask` — 更新已有任务

- 返回 `Result<ScheduledTask>`

`UpdateTaskRequest`（所有字段可选，只更新传入的字段）：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `triggerAtEpochMs` | `Long?` | 否 | 新触发时间 |
| `actionType` | `ActionType?` | 否 | 新事件类型 |
| `actionPayload` | `Map<String, String>?` | 否 | 新事件参数 |
| `label` | `String?` | 否 | 新备注 |
| `isEnabled` | `Boolean?` | 否 | 启用 / 禁用 |
| `repeatIntervalMs` | `Long?` | 否 | 新重复间隔 |

#### 3.2.3 `deleteTask` — 删除任务

- 返回 `Result<Unit>`
- 删除后即使已到触发时间也不再执行，关联的音频缓存一并清理

#### 3.2.4 `getTask` — 查询单个任务

- 传入 `taskId: String`，返回 `Result<ScheduledTask>`

#### 3.2.5 `listTasks` — 列出任务!⁄

- 返回 `List<ScheduledTask>`

`TaskFilter`：

| 枚举值 | 说明 |
|---|---|
| `ALL` | 全部（按触发时间升序） |
| `ENABLED` | 仅已启用 |
| `DISABLED` | 仅已禁用 |

## 4. 事件分发器 · `ActionDispatcher`（内部）

定时任务触发时由 AlarmReceiver 触发，内部路由到对应处理器。外部不直接调用。

```
AlarmReceiver.onReceive()
    → 查出本次触发的任务列表
    → 逐个交给 ActionDispatcher.dispatch(task)
        → actionType == AUDIO_PLAY → AudioPlayActionHandler.invoke(payload)
        → actionType == NOTIFICATION → NotificationActionHandler.invoke(payload)
        → ...
    → 一次性任务自动标记 isEnabled=false
    → 重复任务自动计算下次触发时间并重设 Alarm
```

## 5. 事件参数规范

### 5.1 `AUDIO_PLAY` 参数

| key | 类型 | 必填 | 说明 |
|---|---|---|---|
| `audioBase64` | `String` | 是 | Base64 编码的音频二进制数据 |
| `mimeType` | `String` | 否 | 音频格式，默认 `audio/mpeg` |

> ActionDispatcher 收到后解码 `audioBase64` → `ByteArray`，调用 `AudioPlayer.play()`。

### 5.2 新增事件只需定义自己的 key

例如未来新增 `NOTIFICATION`：

| key | 类型 | 说明 |
|---|---|---|
| `title` | `String` | 通知标题 |
| `body` | `String` | 通知正文 |

定时任务模块代码完全不用改。

---

## 6. 容量 & 限制

| 项目 | 限制 | 说明 |
|---|---|---|
| 任务总数上限 | 100 | 超过时 `createTask` 返回失败 |
| `audioBase64` 上限 | 800 KB | 解码后约 600 KB，约 30 秒 MP3 |
| 最小触发间隔 | 60 秒 | 创建 / 更新时校验 |
| `repeatIntervalMs` 最小值 | 60_000（1 分钟） | |
| Alarm 唤醒精度 | ~ 秒级 | 使用 `setExactAndAllowWhileIdle` |

---

## 7. 错误信息

| 场景 | `Result.failure` 信息 |
|---|---|
| 触发时间不在未来 | `"triggerAtEpochMs must be at least 60s in the future"` |
| 任务不存在 | `"Task not found: {id}"` |
| 任务数超限 | `"Task limit (100) reached"` |
| `audioBase64` 为空或非法 | `"audioBase64 is required for AUDIO_PLAY"` |
| `audioBase64` 超长 | `"audioBase64 exceeds 800KB limit"` |
| Alarm 权限未授权 | `"SCHEDULE_EXACT_ALARM permission not granted"` |
